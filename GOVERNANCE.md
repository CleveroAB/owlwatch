# Governance

owlwatch uses a lightweight maintainer-led governance model.

## Maintainers

The Clevero AB maintainers listed in `.github/CODEOWNERS` are responsible for:

- reviewing and merging contributions;
- maintaining release quality and security boundaries;
- publishing releases and security advisories;
- keeping the documented API and configuration contract stable.

## Decisions

Routine changes are decided through pull-request review. Significant changes to security, deployment, federation, persistence, or public APIs should include a proposal in a GitHub Discussion before implementation. Maintainers seek community input but retain final responsibility for project direction and release decisions.

## Becoming a maintainer

Consistent contributors may be invited by the existing maintainers based on sustained, constructive participation, sound technical judgment, and responsible handling of security-sensitive changes.

## Conduct and security

Participation is governed by `CODE_OF_CONDUCT.md`. Security reports must follow `SECURITY.md` and should never be filed as public issues when they contain exploitable details.
